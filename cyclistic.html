Case Study

Introduction: This case study is based on data from a Chicago based bikeshare company named Cyclistic. This study follows the six data analysis steps: ask, prepare, process, analyze, visualize, act. The raw data can be found here and the post-analysis excel spreasheet can be found here.

1: Ask
Cyclistic has casual riders that pay a fee for each ride in addition to members who pay an annual fee for unlimited rides. The question for this analysis is how do annual members and casual riders use Cyclistic bikes differently? This question's answer can be used to encourage more casual riders to switch to annual members and generate more revenue.

2: Prepare
The data to be analyzed is from .csv file that contains data for Cyclistic trips for January 2022. This is first party source since it comes from directly from Cyclistic. It is a comprehensive list all 103,770 trips for the month of January. To prepare the data for analysis, I converted the file to a .xlsx file.

3: Process
I created column E called "ride_length" by subtracting the start time from end time and changing to the time format. I also created column F called "day_of_week" using the TEXT function on the "started_at" column. I cleaned the data by filtering it to just include rows with rides that lasted 0 or less minutes and deleting them.

6: Act
The top three recommendations based on my analysis:

  1. Advertise memberships as an inexpensive way to get to work.
  2. Offer cheaper membership that only allow the customer to ride on the weekends. This targets the relatively large amount of casual rides that ride on weekends.
  3. Change the casual ride to cost per minute of riding. This insentivises people to get a membership if they would like to ride for a longer time without penalty.
